# 0002 — the corpus form

- **Date:** 2026-07-06
- **Status:** accepted

## Context

The prior form stifled independently of its vocabulary: 49 decision records
compounded in the read path, aspect-oriented files scattered each noun
across five homes, DRY-by-reference built a citation web no file survived
alone, and the corpus enumerated what code already enumerates (predicate
lists, default-contract clauses) — a standing drift liability that ran files
to 3.5× their own budget.

## Decision

Thing-oriented files: one noun, one section, one home. An append-only
`decisions/` log outside the read path, edges one-way (decision → section,
never back). Equal representation: mechanism and instance data live in code;
the corpus never restates them. Hard per-kind budgets. The form applies the
kernel's own principles — file = member, section = nested member, document
classes = kinds, the decision log = provenance — without adopting temper's
machinery: no headers, no gating, until the engine reaches the kernel. If
the model moves again, the form degrades gracefully to conventional
markdown plus a decision log.

## Rejected

Inline decisions (history taxes every reader of the current contract).
Aspect-orientation (a noun change ripples across files). Corpus-enumerated
instances (drift liability). Wiring the deactivated dogfood now (enforcement
resting on an engine that is behind the model it would enforce).
