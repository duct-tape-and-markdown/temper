# 0019 — content is a declared kind fact; a layout has one face

- **Date:** 2026-07-09 · **Status:** accepted

## Context

The founding posture (pre-kernel: genres, equal authoring postures) let typed
members live inside authored documents. The kernel recut kept the machinery —
nested members — but retired the vocabulary without restating the thesis in
kernel nouns; 0012→0018 then retired the read face truthfully but
contingently: no authored host existed, because the pilot never landed. The
occupant arrives — a spec corpus whose laws, invariants, and features are
typed members authored in plain markdown — and the represented binary left
an authored document no standing.

## Decision

A kind declares its **content**: **file** — the body is one verbatim prose
value; the default; what install hoists — or a **layout** — a declared
template over the body's heading tree in exactly three primitives: prose
(verbatim; a region may be an import — a declared reference resolving to a
file's contents, fingerprinted, graphed, refusing when dangling), a field
section (a heading whose span fills a named slot — intent among them), and a
member collection (a heading whose child headings are each one member of a
named kind). A layout admits no syntax beyond markdown's own and has one
face: the reader. The document is the authored home — emit derives its
declaration rows and writes only the lock, never writing, regenerating, or
reaping the document. A collection member's identity is its slugged heading
— an explicit key survives retitling; fingerprints report a rename as a
move, loudly. 0018 is scoped, not overturned: it binds composed hosts,
whose rendering is a projection; a layout document is source, its declared
layout the typed surface its members are declared on.

## Rejected

- The bidirectional frame — writer and reader from one layout, movable
  postures: a write face with no author, the no-occupant machinery 0018
  demolished once.
- A general format-template grammar: what does not fit the three primitives
  is two kinds, or it is prose.
- Mandated per-entity IDs (an attribute dialect markdown lacks) and
  per-entity fences (the genre-era spelling): declaration lives on the kind.
- Shipping a spec kind as a built-in: Reach's one-corpus bound stands; this
  repo declaring its own spec kind is a customer act, and the demo.
- Collection-grain imports: file children are nesting's existing job.

## Consequences

`intent.md` recuts its center (the documents that program agents) and gains
the read-or-written invariant; `representation.md`'s kind gains the content
fact, nesting scoping "never mined" to composed hosts; `pipeline.md`'s
emit/lock/install acknowledge governed sources; `distribution.md`'s demo
becomes this corpus governing itself. Code: the content fact, the layout
reader (the sections substrate ships), prose-import resolution; coverage
clauses ride the existing algebra — degree over selections.
