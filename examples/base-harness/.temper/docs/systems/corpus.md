The documentation corpus is this repository's declared intent: the systems,
flows, decisions, and terms under `docs/` are typed members of declared
kinds, authored in the harness program and rendered here by `emit`. A
document is never a description trailing the implementation; it is the
specification the implementation answers to, and — the glossary excepted —
never the authored home: edit the owning module under `.temper/docs/`.

## Purpose

Hold the repository's intent in typed, addressable, gated form. A flow's
participants and a superseded ruling's successor are typed fields resolved
as edges; a claim of requirement coverage is the member's own `satisfies`
declaration; the rendered markdown is a projection whose hand-edit is
drift.

## Invariants

### Spec authoritative

Change enters the corpus first. Code follows via reconciliation; a change
that lands in code with no corpus home is drift to surface, never truth to
transcribe backwards.

### Declared never mined

Structure exists because an author declared it on a typed surface: a field
on a member, an entry in an edge section, a member in a collection. No
tool derives model structure by pattern-matching prose.

### Return paths are declarations

Evidence against the corpus (field reports, implementation pain) enters
through an authored surface — a proposed decision, an open question —
never by editing downstream artifacts first.
